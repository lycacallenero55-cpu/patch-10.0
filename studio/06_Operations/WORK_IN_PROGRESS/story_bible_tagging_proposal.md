- **Author:** ② Lore Department, CYCLE-0001.
- **Target file:** `manuscript/bible/story_bible.md` (read-only for this proposal — not edited).
- **Why this exists:** `SOURCE_OF_TRUTH.md` classifies `story_bible.md` as "MIXED: FRANCHISE_CANON sections + HISTORICAL_RECORD/SUPERSEDED production notes … do not treat wholesale as active" and assigns the section-tagging work to TASK-0004. Handbook §5 requires every authoritative document to be classified by section before it is relied upon. This proposal does that classification without touching the source file, per the DEC-0008 block on substantive canon rewrites (tagging/labeling is explicitly legal under Cycle-1 scoping; applying the tags as literal in-file markup is not, until the creator approves this proposal).
- **Tag vocabulary used** (per work order): `FRANCHISE_CANON` (active story truth) · `HISTORICAL_RECORD` (true about production history, not story law) · `SUPERSEDED` (deprecated guidance — old engines, old styles, old shot-length rules) · `PRODUCTION_NOTE` (operational/how-we-made-it detail that isn't story law and isn't yet historical).
- **Method:** each heading in `story_bible.md`, in document order, gets exactly one proposed tag plus a one-line rationale. Where a single section mixes types internally (common in the later production-log sections), the proposal flags the split explicitly rather than forcing one tag over mixed content — a future in-file tagging pass should sub-divide those sections rather than tag them monolithically.

## Cross-cutting observations (not new tags — findings surfaced by doing this pass)

1. **The file's own title line claims blanket "(canon)" status**, which is exactly the inaccuracy SOURCE_OF_TRUTH.md and TASK-0004 exist to correct. This proposal does not recommend removing that word from the title (that would be an edit to the protected file, out of scope this turn) — it recommends that a future approved tagging pass either retitle the header or add a prominent "see per-section tags below" caveat directly under the title.
2. **Three sequential art-generation iterations are effectively an unbroken chain of supersession** (v1 reference set → v2 supporting-cast set → EP.01 v2 rebuild → ART v3 FINAL LOCK → EP.01 v3 Part 1 → strip doctrine v5/Part 1 v5 → Part 2 anchors/overnight run). 3. **No FRANCHISE_CANON section in story_bible.md was found to contradict prose canon** (`00_prologue.md`, `01_the_ninth_world.md`) or `Master_Project_Bible.md`. This is a positive continuity finding: the mixed-file problem TASK-0004 addresses is a *labeling* problem (production history interleaved with active canon in one long file), not a *factual contradiction* problem. No new OPEN_QUESTIONS entries or CONTINUITY_LEDGER conflicts were generated by this pass.
4. **One embedded UNDECIDED item was found and correctly left undecided:** the Im Dan-bi first-death candidacy inside the Cast section (row 8 above) maps to Q-01 in `OPEN_QUESTIONS.md`. No resolution attempted here, per work-order constraint.
5. **This proposal recommends, but does not perform, splitting mixed sections** (rows 15, 20, 27 above) into sub-tagged pieces. That splitting is itself a (minor) edit to the protected file and should wait for creator approval alongside the rest of this proposal.
