## Cross-cutting usage rules (restated from `studio/00_STUDIO_RULES.md` §4, not duplicated in full — read that file for the complete list)

- Engine: `openai/gpt-image-2` (quality high) for art; Veo 3.1 for motion. NB2/Pro are deprecated for this project's art (see the Art_Direction.md appendix for why).
- Every art call needs `inputImages` = [environment plate] + [character master(s)] (+ a real-PMU style-anchor tile when drift has appeared in a recent batch) — restate hair color/outfit/scar-side/weapon-hand in the prompt text even when a reference image is attached.
- Flat-2D law applies to every generation regardless of which appendix's exemplar you're adapting: ink outlines first, flat matte fills, ≤2–3 hard cel-shadow tones, cross-hatch shading, no 3D volume/gloss/blur/photorealism/airbrush.
- All lettering is HTML overlay (readers) or Pillow cards (video) — never baked into generated art, in any appendix's pattern.
- The sealed twist file (`manuscript/bible/twist_bank_PRIVATE.md`) is never surfaced in any prompt, in any appendix, for any reason — this rule has no appendix-specific exception.

## What this file deliberately does NOT do

- It does not reproduce any prompt text verbatim (that would violate the single-source rule the header states).
- It does not judge whether an appendix's guidance is still current — if `Image_Workflow_QA.md`'s or `Video_Workflow.md`'s main-body recipe has since diverged from its own appendix's older exemplars, that is a ③-owned question, not resolved here. (No such divergence was found during this cycle's build of this index, but ④'s charter is to point, not to audit ③'s internal consistency — that is ⑤'s lane.)
- It does not add new prompt guidance of its own. Every rule restated in the "Cross-cutting usage rules" section above is copied verbatim in spirit from `studio/00_STUDIO_RULES.md` §4 for at-a-glance convenience, but that file remains the actual source — if this section and that file ever disagree, `00_STUDIO_RULES.md` wins.
